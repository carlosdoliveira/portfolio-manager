    Comparação: Antes vs Depois¶

    ❌ ANTES (Problema Potencial)¶

    Usuário poderia pensar que operações em mercados diferentes 
criam posições separadas:

PETR4 (Mercado à Vista)      → 100 ações
PETR4 (Mercado Fracionário)  →   5 ações
                              ─────────────
                                 ??? Total?

    ✅ DEPOIS (Solução Clara)¶

    Sistema mostra explicitamente que há consolidação:

┌─────────────────────────────────────┐
│ Posição Atual: 105 ações           │
│ Consolidada (todos os mercados)    │ ← Nota explicativa
└─────────────────────────────────────┘

Com drill-down disponível:
  └─ Resumo por Mercado (informativo)
      ├─ MERCADO A VISTA: 100 ações
      └─ MERCADO FRACIONARIO: 5 ações

    Benefícios da Abordagem¶

    Aspecto Antes Depois
    Clareza Implícito Explícito com nota visual
    Transparência Apenas total Total + breakdown por mercado
    Auditoria ✅ Preservada ✅ Preservada + visível
    Educação Usuário precisa deduzir Sistema explica
    UX Básica Profissional e informativa

    Exemplos de Uso¶

    Caso 1: Investidor Iniciante¶

    👤 "Comprei 100 PETR4 no mercado à vista e 5 no fracionário. 
    Quantas ações eu tenho?"

💡 Sistema mostra:
   - Card: "Posição Atual: 105 ações (consolidada)"
   - Resumo por mercado explica a origem

    Caso 2: Investidor Experiente¶

    👤 "Quero saber se minhas compras no fracionário têm preço 
    médio diferente do mercado à vista"

💡 Sistema permite:
   - Ver posição consolidada
   - Drill-down no resumo por mercado
   - Análise detalhada na tabela de operações

    Caso 3: Auditoria¶

    👤 "Preciso verificar se todas as minhas operações da B3 
    estão corretas"

💡 Sistema oferece:
   - Tabela completa com coluna "Mercado"
   - Cada operação preservada como importada
   - Totais conferem com posição consolidada

    Arquitetura Event-Based¶

    ┌──────────────────────────────────────┐
│ EVENTO 1 (Imutável)                 │
│ ─────────────────────────            │
│ Date:   2026-01-01                  │
│ Type:   COMPRA                      │
│ Market: MERCADO A VISTA             │
│ Qty:    100                         │
│ Price:  R$ 30.00                    │
└──────────────────────────────────────┘
              +
┌──────────────────────────────────────┐
│ EVENTO 2 (Imutável)                 │
│ ─────────────────────────            │
│ Date:   2026-01-05                  │
│ Type:   COMPRA                      │
│ Market: MERCADO FRACIONARIO         │
│ Qty:    5                           │
│ Price:  R$ 31.00                    │
└──────────────────────────────────────┘
              ║
              ▼
┌──────────────────────────────────────┐
│ ESTADO DERIVADO (Calculado)         │
│ ────────────────────────             │
│ Posição:     105 ações              │
│ Investido:   R$ 3.155,00            │
│ Preço Médio: R$ 30.05               │
└──────────────────────────────────────┘

    Princípio: - Eventos são a fonte da verdade (immutable) - Estado é derivado em runtime (calculated) - Consolidação é parte da derivação (transparent)

    Conclusão¶

    A consolidação de mercados é uma feature, não um bug.

    O sistema foi projetado para: 1. ✅ Armazenar a realidade (operações com mercado) 2. ✅ Calcular o estado atual (posição consolidada) 3. ✅ Mostrar ambos de forma clara (transparência)

    Esta abordagem respeita os princípios do projeto e oferece a melhor experiência para o usuário.
